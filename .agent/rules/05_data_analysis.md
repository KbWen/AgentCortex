# RULE: DATA ANALYSIS & REPORTING (05)

## ⚖️ MODEL SELECTION

- **FLASH**: Use for simple statistics (Mean/Median), JSON format validation, or basic log filtering.
- **PRO**: Use for correlation analysis, root-cause deduction from massive logs, or generating strategic business reports.

## 📋 ANALYSIS PROTOCOL

1. **Audit**: Verify data integrity and missing values.
2. **Explore**: Identify trends, outliers, and patterns.
3. **Reporting**:
   - Use `eda_reporting` skill for structure.
   - Always include "Actionable Recommendation" at the end.

## 🚫 RESTRICTIONS

- Never hypothesize without citing specific data points.
- Do not assume correlation equals causation.
