# Docs Update Workflow

1. `/spec`：定義文件受眾、成功條件與更新範圍。
2. `/plan`：列出受影響文件（README / CHANGELOG / 指南）。
3. `/implement`：同步更新內容，避免文件互相矛盾。
4. `/review`：檢查術語一致性與平台指引一致性。
5. `/retro`：沉澱可複用模板與後續優化項。
